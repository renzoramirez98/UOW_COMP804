Task 3

Name Web Attacks
SQL Injection (SQLi)


Source: https://blogs.zeiss.com 		SQL Injection is a code injection technique used to attack data-driven applications. It involves inserting malicious SQL statements into application queries to manipulate the database. This attack can lead to unauthorized data access, data modification, or even complete system takeover. SQL Injection remains one of the most dangerous vulnerabilities in web applications.

Learn more about SQL Injection
Cross-Site Scripting (XSS)


Source: https://www.veracode.com 		Cross-Site Scripting (XSS) attacks involve injecting malicious scripts into trusted websites. These scripts execute in the victim's browser, potentially stealing sensitive information, hijacking user sessions, or defacing websites. XSS can be persistent (stored on the target servers) or reflected (contained in a link), making it a versatile and dangerous attack vector.

Learn more about Cross-Site Scripting
Cross-Site Request Forgery (CSRF)


Source: https://encrypted-tbn0.gstatic.com 		Cross-Site Request Forgery tricks the victim into submitting a malicious request. It inherits the identity and privileges of the victim to perform an undesired function on the victim's behalf. CSRF attacks specifically target state-changing requests, not theft of data, since the attacker has no way to see the response to the forged request.

Learn more about Cross-Site Request Forgery
Remote Code Execution (RCE)


Source: https://cdn.prod.website-files.com 		Remote Code Execution is a cyber attack where an attacker can remotely execute commands on someone else's computing device. These attacks are often the result of malicious malware downloaded by the victim and can happen regardless of the device's geographic location. RCE attacks can lead to data theft, loss, or encryption, and even complete system takeover.

Learn more about Remote Code Execution
Distributed Denial of Service (DDoS)


Source: https://infosavvy.home.blog 		A Distributed Denial of Service attack is an attempt to make an online service unavailable by overwhelming it with traffic from multiple sources. Attackers build networks of infected computers (botnets) to launch these attacks. DDoS attacks can target various components of a network connection, aiming to exhaust server resources and bandwidth.

Learn more about Distributed Denial of Service
Man-in-the-Middle (MitM)


Source: https://beaglesecurity.com 		In a Man-in-the-Middle attack, the attacker secretly relays and possibly alters the communications between two parties who believe they are directly communicating with each other. The attack allows the perpetrator to intercept, send and receive data meant for someone else, or not meant to be sent at all, without either outside party knowing until it is too late.

Learn more about Man-in-the-Middle attacks
Directory Traversal


Source: https://cdn.acunetix.com 		Directory Traversal (also known as path traversal) is an HTTP exploit which allows attackers to access restricted directories and execute commands outside of the web server's root directory. An attacker may manipulate a URL in such a way that the web site will execute or reveal the contents of arbitrary files anywhere on the web server.

Learn more about Directory Traversal
Broken Authentication and Session Management


Source: https://www.sitelock.com 		Broken Authentication and Session Management refers to weaknesses in handling user authentication and managing active sessions. These vulnerabilities can allow attackers to compromise passwords, keys, or session tokens, or exploit other implementation flaws to assume other users' identities temporarily or permanently. This can lead to unauthorized access to sensitive data and functionality.

Learn more about Broken Authentication and Session Management
Security Misconfiguration


Source: https://media.licdn.com 		Security Misconfiguration is a common issue where security settings are defined, implemented, and maintained as defaults or in an ad hoc manner. Good security requires having a secure configuration defined and deployed for the application, frameworks, application server, web server, database server, and platform. This includes keeping all software up to date, including all code libraries used by the application.

Learn more about Security Misconfiguration
Insecure Deserialization


Source: https://miro.medium.com 		Insecure Deserialization occurs when untrusted data is used to abuse the logic of an application, inflict a denial of service (DoS) attack, or even execute arbitrary code. It happens when an application receives hostile serialized objects and deserializes them. This vulnerability can result in remote code execution, which is one of the most serious attacks possible.

Learn more about Insecure Deserialization
File Inclusion Attacks


Source: https://www.thesecuritybuddy.com 		File Inclusion Attacks allow an attacker to include a file, usually exploiting a "dynamic file inclusion" mechanisms implemented in the target application. The vulnerability occurs due to the use of user-supplied input without proper validation. This can lead to code execution on the web server, data theft, or denial of service, depending on the severity and the files the attacker is able to leverage.

Learn more about File Inclusion Attacks
